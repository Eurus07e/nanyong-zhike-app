# 安全说明

南雍知课会处理南京大学统一身份认证和个人学业数据。请不要在公开 Issue、截图、日志或复现文件中包含真实密码、CASTGC、Cookie、学号、成绩或课表。

## 报告安全问题

发现安全问题时，请通过 [yuxuanshu@smail.nju.edu.cn](mailto:yuxuanshu@smail.nju.edu.cn) 私下联系维护者，并提供不含真实凭据的最小复现步骤。在维护者确认问题并完成修复前，请不要公开漏洞细节。

当前正式版本和仓库默认分支接受安全问题报告。旧版本可能不再单独修复，建议先升级到 [最新 Release](https://github.com/Eurus07e/nanyong-zhike-app/releases/latest) 后再确认问题是否仍然存在。

## 数据保护

- 密码仅用于发起当次南京大学统一身份认证，不会写入数据库、浏览器存储或日志。
- 学校认证票据在本机使用随机密钥加密；浏览器只持有随机的 `HttpOnly` 会话 Cookie。
- 服务端按学号保存最近一次成绩、排名、课表、培养方案、五育总览、五育活动和第二课堂快照，用于先显示上次结果、后台刷新和新成绩比较；所有快照均使用 `APP_SECRET` 派生密钥加密，浏览器只在当前会话内存中使用启动快照，不持久保存学业数据。为完成通识、英语和体育认定，程序会按学期批量查询已通过课程的官方分类字段。
- 备忘录正文持久保存在运行服务的 SQLite 数据库中并按学号隔离；共享部署的维护者能够接触该数据库，因此只能在可信部署中记录私人内容。
- 自愿支持由支付宝直接处理；本站不接入支付接口、回调或交易数据库。收款码会公开脱敏收款昵称，付款前应在支付宝内核对收款方。
- 生产部署必须使用 HTTPS 和至少 32 个字符的独立随机 `APP_SECRET`，并限制 `.env` 与持久化数据卷的访问权限。

## 排名服务的上游限制

排名与平均学分绩来自目前只支持 HTTP 的南京大学交换生系统。本地发行包为保持功能完整会在登录后进入“学业概览”时自动访问该系统；公网部署默认关闭此链路。

该上游连接无法提供 HTTPS 的机密性与完整性保护，认证后的单次票据、交换系统会话及返回数据可能被不可信网络观察或篡改。不要在不受信任的远程部署环境中设置 `ALLOW_INSECURE_EXCHANGE_SYSTEM=true`。这一限制来自学校上游系统，无法通过南雍知课自身的 HTTPS 配置消除。
