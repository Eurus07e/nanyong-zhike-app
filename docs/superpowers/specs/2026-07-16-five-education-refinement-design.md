# 五育总览精修与我的活动设计说明

日期：2026-07-16  
目标版本：v1.2.0  
状态：方案 A 已批准，待实现

## 范围

只修改“校园服务 > 五育系统”页面及其专属接口、类型、测试和样式，不调整其他既有页面或全站设计令牌。页面继续保持只读，不报名、取消、评价或修改五育系统记录。

## 已确认的真实数据

- 总览继续来自 `GET /dztml/wdwy`。
- 年级平均活动数按年级活动总数除以年级人数计算，保留一位小数，不强制取整。
- “我的活动”菜单实体为 `rt.rt-wdhd`。客户端从登录后的 `/dztml/ctx` 解码菜单并定位对应菜单 ID，不在代码中固定用户或票据。
- 当前学年学期由 `/dztml/rt/wdhd?.me=...` 页面给出；活动列表来自只读接口 `/dztml/rt/wdhd/ajaxList`。
- 已用真实会话验证当前学期返回 8 条活动记录。

## 页面结构

### 标题区

- 左侧保留“我的五育”及简短说明。
- 右侧仅保留一个“打开南京大学五育系统”图标按钮和一个刷新按钮。
- “数据来自南京大学五育系统 · 查询于 …”紧接在刷新按钮下方，右对齐。
- 删除页面底部重复的“打开原系统”入口和来源页脚。

### 核心总览

- 摘要指标增大字号，在不拥挤的情况下提高标签、注释、图例和卡片标题的可读性。
- 雷达图与精确数字表保持同一卡片：左侧雷达图放大，右侧数字表收窄；手机端上下排列。
- 成长模块与劳育构成参考官网“成长状态”信息层级，但沿用南雍知课紫色、浅灰、白色和统一圆角，不照搬官网彩虹色块。
- 劳育卡片标题区增加“查看学习导引图”按钮，弹层展示用户提供的《南京大学本科生劳动教育学习导引图》；支持关闭、背景点击关闭和 Escape 关闭，图片可在窄屏完整缩放。
- 删除“我的兴趣”。学期评价作为独立的紧凑信息块保留。

### 我的活动

- 总览下方新增“我的活动”板块，初始展示当前学年学期全部活动。
- 提供关键词、活动状态和排序控件；筛选与排序在当前已加载记录中即时完成。
- 桌面端使用紧凑表格，核心列为活动名称、类型/模块、发起单位、活动时间、地点、审核/认定状态和成绩/时长。
- 点击整行或“查看详情”展开详情面板；手机端直接使用可展开卡片，避免横向挤压。
- 详情尽量展示上游提供且有实际参考价值的字段：英文名、活动类型、所属模块、劳动类型、组织单位、负责人、联系方式、报名时间段、本人报名时间、活动时间段、地点、报名方式、招募人数、活动介绍、考核办法、审核状态、认定状态、成绩、录入/认定时长和评价状态。
- 不返回或显示学号、姓名、微信号、用户手机号、身份证件、CASTGC、service ticket、五育 Cookie、内部操作者编号及无参考价值的空字段。
- 日期解析失败时显示“时间待上游确认”，不显示 `Invalid date`。

## 数据与安全

- 后端新增独立的活动标准化函数和只读活动查询方法，继续使用临时 CookieJar；请求结束后不保留五育站点 Cookie。
- 活动响应仅包含白名单字段，任何嵌套上游对象先标准化再返回，避免意外泄露个人字段。
- 前端沿用 5 分钟内存缓存；刷新按钮同时强制刷新总览与活动。
- 认证过期映射为 401，上游故障与数据格式异常映射为 502，错误文案不包含票据、查询参数或原始响应。

## 验收标准

- 桌面与 390px 手机视口无横向溢出，雷达图标签和活动详情不裁切。
- 来源时间位于刷新按钮下方，页面下方没有第二个原系统跳转。
- 劳育导引图按钮可打开并完整查看参考图。
- 年级平均保留一位小数，“我的兴趣”不存在。
- 当前真实账号可看到 8 条当前学期活动；筛选、排序和详情展开正常。
- 页面主内容及 API 响应不包含学号、姓名、微信号、CASTGC 或五育 Cookie。
- 后端全量测试、前端单测、Lint、生产构建与浏览器桌面/手机视觉验收通过。
