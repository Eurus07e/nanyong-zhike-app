# 培养方案修复与轻量备忘录设计

## 目标

修正培养方案中课程池、固定课程汇总与毕业要求混用导致的学分/门数错误，并确保学业概览始终使用用户本人专业方案。随后在不增加独立服务的前提下，加入与南雍知课现有界面一致的中文备忘录。

## 培养方案口径

- 顶层四类与总学分以方案详情 `XDYQ` 文本中的毕业要求为权威来源。
- 课程组的应修门数优先取 `ZSXDMS`，应修学分优先取 `ZSXDXF`。
- 固定课程节点在原始汇总缺失时，从该节点实际课程列表计算课程数和学分。
- 选修节点的课程列表代表“可选课程池”，课程池总学分不能显示为应修学分。
- 父节点只有在全部直接子项都能得到同一应修口径时才递归汇总；否则使用顶层文本要求，或仅显示子模块数量。
- 节点卡展示应修口径；节点弹窗单独显示课程池数量、课程池总学分及筛选后的课程清单。
- 分类节点详情聚合所有后代课程叶子，去重后展示；单个课程接口失败时保留其他结果并给出局部提示。
- 学年模式区分“固定课程安排”和“选修课程池”，不把池内所有课程合计成学生必须修读的门数/学分。

统计与树遍历提取为无 React 依赖的纯 TypeScript 模块，培养方案结构图、节点详情和学业概览共用。课程匹配继续保留大学体育等效匹配和“未直接匹配不等于缺课”的说明。

## 本人专业隔离

学业概览每次根据 `/api/academic/profile` 取得年级、专业和院系，再从该年级方案中选择本人主修方案。培养方案浏览页的选择只作为浏览偏好保存在单独键名中，不成为学业概览的数据源，也不会污染本人方案缓存。

## 轻量备忘录

不部署完整 `usememos/memos` 服务，不增加 Go、容器或额外账户体系。现有 FastAPI 与 SQLite 增加单表和同源 API，每条数据以当前统一身份认证会话中的 `username` 隔离。

首版功能：

- 新建纯文本备忘录；支持多行文本和从内容中识别 `#标签`。
- 编辑、删除、置顶/取消置顶。
- 按正文或标签搜索；按置顶优先、更新时间倒序展示。
- 中文空状态、错误状态和删除确认。
- 左侧导航新增“备忘录”，页面按访问后保活，但每次写操作以服务端返回为准。

不做附件、评论、通知、多人协作、外部同步、复杂权限、Markdown 富文本和独立 Memos 数据兼容层。数据库正文限制 10,000 字，标签由服务端重新提取，避免前后端口径不一致。

## API 与数据

SQLite `memos` 表包含：`id`、`username`、`content`、`tags_json`、`pinned`、`created_at`、`updated_at`。索引覆盖用户时间线和用户置顶排序。

- `GET /api/memos?q=`：返回当前用户的时间线。
- `POST /api/memos`：新建。
- `PATCH /api/memos/{id}`：修改正文或置顶状态。
- `DELETE /api/memos/{id}`：删除并返回 `204`。

不存在或属于其他用户的 ID 统一返回 `404`，防止暴露其他用户数据。所有写接口沿用现有同源与会话保护。

## 错误处理与验证

- 培养方案课程批量请求使用 `Promise.allSettled`，认证过期仍立即退出，其他上游错误作为局部失败展示。
- 备忘录数据库操作放在线程中执行，不阻塞 FastAPI 事件循环。
- TypeScript 领域逻辑使用 Node 内置测试器，不增加测试框架依赖。
- Python 覆盖数据库 CRUD、用户隔离、标签提取、输入校验和 API 鉴权。
- 最终执行前端 lint/build、全部 pytest，并在本地真实浏览器验证登录页资源、培养方案两种模式、本人概览隔离、备忘录增删改查、桌面与移动布局及控制台。

## 发布边界

本轮先在 `http://127.0.0.1:8000` 交给用户验收。未得到用户确认前不推送、不创建 Release。后续发布版本按 `v1.1.0` 处理，并继续生成 Windows、macOS、Linux 原生包。
