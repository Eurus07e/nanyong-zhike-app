# 五育总览设计说明

日期：2026-07-16
目标版本：v1.2.0
状态：已批准并实现

## 目标与范围

v1.2.0 首先实现“我的五育”只读总览。页面复用南雍知课已有统一身份认证会话，从南京大学五育系统读取真实个人数据，以克制的图表展示活动分布、成长模块、劳育构成、兴趣和学期评价。

本阶段不修改五育系统中的兴趣、评价或其他记录，不报名活动，不生成自助打印码，不在本站长期保存五育个人数据。第二课堂个人记录另行设计和实现。

## 已验证的上游系统

- 五育入口：`https://ndwy.nju.edu.cn/dztml/`
- CAS service：`https://ndwy.nju.edu.cn/dztml/`
- 数据接口：认证后 `GET https://ndwy.nju.edu.cn/dztml/wdwy`
- 认证方式：现有加密 CASTGC 可向南京大学统一身份认证换取一次性 service ticket；五育系统随后建立自己的 `SESSION`、`JSESSIONID` 等站点会话。
- 上游页面技术：Vue、Axios、ECharts。

2026-07-16 已使用本机有效南雍知课会话完成只读验证。验证过程中未输出或保存 CASTGC、service ticket、Cookie 值、学号和个人指标值。

## 上游数据与标准化模型

上游响应的个人数据集中在 `extend`：

- `dbzt`：个人五育状态。
  - `dhds`、`zhds`、`thds`、`mhds`、`lhds`：德、智、体、美、劳活动数。
  - `mksc`：JSON 字符串，按成长模块 ID 保存实际时长。
  - `zsc`：劳育总时长。
  - `mks`：个人兴趣类别 ID 列表。
  - `xh`、`wdbyy` 等身份或内部字段不得返回前端。
- `njhds`：同年级五类活动总量和年级标识。
- `njrs`：各年级人数，用于计算同年级平均活动数。
- `mkList`：成长模块定义，已验证包含基础实践、学科实践和综合实践，以及达标时长 `dbsc`、劳育显示目标 `fdsc`。
- `dpj`、`ypj`：待评价数、已评价数。
- `wysm`：五育规则说明。
- `dyks`、`dyjs`、`sfkdy`：打印开放时间和可打印状态；首版不在总览中执行打印操作。
- `wdhdMe`：上游活动详情内部标记；不得透传前端。

本站后端对外返回稳定的 `FiveEducationOverview`：

同年级平均值以 `njhds` 对应维度总量除以 `njrs[njhds.nj]` 计算，保留一位小数；雷达图最大值按本人值和同年级平均值逐维动态确定，活动数不转换成没有官方依据的百分比。

成长模块的达成状态严格使用上游规则：实际时长必须为正数，并且 `actualDuration >= dbsc`；没有实际时长时判定为未达成。当 `dbsc` 为零时显示系统判定结果，不绘制伪造的百分比进度。劳育构成使用 `mksc` 的实际时长和 `mkList.fdsc` 的显示目标；`fdsc >= 99` 时不显示目标值，与上游页面行为一致。

评价完成率为 `ypj / (dpj + ypj)`；总数为零时返回 0，而不是 `NaN`。所有合法零值必须显示为 `0`，不能被当作缺失数据。

## 后端结构

新增独立的 `FiveEducationClient`，不把五育逻辑塞入现有 eHall 或交换生系统客户端。

客户端职责：

1. 接收当前 `Session.castgc`，只在内存中使用。
2. 向固定 CAS service 换取一次性票据。
3. 只允许重定向到 `authserver.nju.edu.cn` 和 `ndwy.nju.edu.cn`。
4. 使用五育站点临时 Cookie 请求 `/dztml/wdwy`。
5. 校验 JSON 响应和必要字段，转换为稳定模型。
6. 请求结束后释放 CookieJar，不把五育 Cookie 写入数据库或磁盘。

新增认证接口：

```text
GET /api/five-education/overview
```

接口通过现有 `current_session` 依赖获取会话。CAS 失效返回 401；五育系统不可用返回 502；响应字段异常返回 502 和面向用户的稳定错误文案。错误信息不得包含上游 URL 查询参数、票据、Cookie 或响应正文。

首版不把五育数据加入 `portal_snapshots`。前端只使用当前页面内存缓存，默认有效 5 分钟；点击刷新时绕过缓存重新读取。

## 前端信息结构

入口保持在“校园服务 > 五育系统”。采用用户确认的“平衡总览”方案，与现有 Apple 风格、统一灰色、8px 圆角和紫色强调色一致。

### 顶部

- 标题“我的五育”。
- 简短说明“德、智、体、美、劳活动与成长模块总览”。
- 右侧刷新按钮。
- 不增加英文眉题或重复二级说明。

### 三项摘要

- 活动总数：五类个人活动数合计。
- 劳育总时长：`dbzt.zsc`。
- 学期评价完成率：已评价数 / 总数，同时显示精确计数。

### 主图表

- 左侧为五维雷达图，比较“我的活动数”和“同年级平均”。
- 雷达图使用纯 SVG 和现有 React/CSS，不引入 ECharts、Recharts 等新运行时依赖，保持安装包轻量。
- 图表旁或图表下方必须提供五维精确数值，不能只靠图形或颜色传达信息。
- 紫色表示本人，统一浅灰表示同年级平均。

### 成长模块

- 右侧展示基础实践、学科实践、综合实践。
- 每项显示实际时长、达标时长和上游判定状态。
- 有正数达标时长时显示进度条；零阈值时只显示“已达成/未达成”和系统规则，不显示虚假百分比。

### 次级信息

- 劳育构成：三类实践实际时长、可用的目标时长和总时长。
- 我的兴趣：把 `mks` 映射为德智体美劳标签，只读显示，不提供修改交互。
- 学期评价：已评价数、总数和完成率。
- 底部显示数据来源、当前查询时间和“打开原系统”安全外链。
- 首版不在本站执行电子成绩单导出、自助打印码或活动详情内部跳转。

## 页面状态

- 首次加载：使用与现有页面一致的轻量骨架，不显示假数据或全零占位。
- 正常状态：展示完整图表和精确值。
- 合法全零：展示零值、空雷达基线和“暂时没有活动记录”，不误报错误。
- 部分字段缺失：可安全推导的区块继续显示，缺失区块显示“数据暂不可得”；同时保留刷新和原系统入口。
- CAS 过期：沿用全站 401 处理，返回登录界面。
- 上游故障：显示“南京大学五育系统暂时不可用”，不展示旧的伪造数据。
- 移动端：摘要改为单列；雷达图与成长模块上下排列；精确数值保持可读，不产生横向页面滚动。

## 隐私与只读边界

- 不记录或返回 CASTGC、service ticket、五育 Cookie、学号、`wdhdMe`。
- 不把上游原始 JSON、HTML 或内部错误正文写入日志。
- 不调用 `updateSearchMks`、`print`、成绩单导出等修改或生成型接口。
- 不长期保存五育个人数据，浏览器仅保留 5 分钟内存缓存。
- 隐私说明补充“五育数据按需查询、短期内存缓存、不写入个人档案”。

## 测试与验收

后端：

- 使用脱敏固定响应测试字段映射、五维平均值、模块达成、零阈值、评价率和合法零值。
- 测试 CAS service 和重定向域名白名单。
- 测试 CAS 过期、上游 5xx、非 JSON、缺失字段和超时。
- 测试响应不包含 `xh`、`wdhdMe`、票据或 Cookie。

前端：

- 测试正常、全零、部分缺失、加载、认证过期和上游故障状态。
- 测试刷新绕过缓存，普通切换在 5 分钟内不重复请求。
- 测试雷达图和精确数值使用相同数据源。
- 测试综合实践零阈值不产生除零或虚假百分比。

浏览器验收：

- 桌面 1440×1000、参考大屏 2048×1114、手机 390×844。
- 页面无裁切、横向溢出、控制台应用错误或无响应按钮。
- 刷新后数据更新时间变化；“打开原系统”指向固定 HTTPS 五育入口。
- 图表颜色、圆角、按钮、间距和校园服务现有设计语言一致。

## 完成标准

用户登录一次南雍知课后，打开“校园服务 > 五育系统”即可读取真实五育总览；页面能准确展示五维本人/年级对比、成长模块、劳育、兴趣和评价数据，并在异常或零数据下给出可靠状态。实现不增加重量级图表依赖，不写入五育个人档案，不执行任何上游写操作。
